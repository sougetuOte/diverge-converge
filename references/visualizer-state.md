# Visualizer: プロセスステートビュー実装ガイド v2

## 目的

ユーザーが「今、二重ループのどこにいるか」を一目で把握できるようにする。
認知負荷の管理の一環。現在地が分かれば「次に何が起きるか」の予測が立つ。

## レイアウト: 入れ子構造

外側ループが全体を囲み、内側ループがその中に配置される入れ子構造。

```
┌─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┐  ← outer loop（灰色破線）
  outer loop (cycle N)
  図示化で空欄を発見したら、再び内側ループへ

  ┌───────────────────────────────────────┐    ← inner loop（色付き背景）
  │ inner loop (iteration M)              │
  │                                       │
  │ [探索] → [反映] → [発散]  → [図示化]  │
  │   ↑__________________________|  iter+1│
  └───────────────────────────────────────┘
           ↑                        │
           └────── cycle+1 ─────────┘
  cycle 2, 3, ... (空欄を起点に再び内側ループ)
└─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┘
```

## ラベルルール

- **全てのユーザー向けラベルは日本語**: 探索・反映・発散・図示化
- 通過済みのモードは「通過」と表示（「完了」は一方向に見えるため使わない）
- 現在のモードは「現在」と表示
- 未着手のモードは「次へ」と表示

## ノードの状態表示

### 外側ループ（大枠）

| 要素 | スタイル |
|------|---------|
| 枠線 | 灰色破線（var(--color-border-tertiary)、stroke-dasharray="8 4"） |
| 角丸 | rx=20（大きめ、包含を示す） |
| ラベル | 上部に「outer loop (cycle N)」、直下に「図示化で空欄を発見したら…」の説明 |
| 下部 | 破線枠＋テキストで「cycle 2, 3, ...」の将来を示唆 |

### 内側ループ（内枠）

| 要素 | スタイル |
|------|---------|
| 背景 | c-teal系（薄い色付き背景で囲みを強調） |
| 枠線 | 実線0.5px |
| 角丸 | rx=14 |
| ラベル | 上部に「inner loop (iteration M)」 |

### 各モードノード

| 状態 | 枠線 | 背景 | テキスト |
|------|------|------|---------|
| 通過 | 実線0.5px、c-green | c-green | 「通過」 |
| 現在 | 太線1.5px、c-teal | c-teal | 「現在」 |
| 次へ | 破線、c-gray | c-gray | 「次へ」 |

### 循環の矢印（2本）

1. **内側の戻り（iteration +1）**: 発散→探索への破線矢印、内側ループ内に収まる。ラベル「iteration +1」
2. **外側の戻り（cycle +1）**: 図示化→探索への矢印、外側ループの下を回る。色は緑系、ラベル「空欄発見 → cycle +1 で内側ループ再開」

## インタラクション

- 「図示化」ノードはクリック可能: `sendPrompt('そろそろ図示化してみよう。マンダラチャートに落としてみて')`
- 「もう少し探索を続ける」ボタン（任意）: `sendPrompt('もう少し探索を続けたい')`

## 更新タイミング

- セッション開始時（初期状態の表示）
- モード遷移時（探索→反映、反映→発散、等）
- 外側ループ遷移時（図示化突入、再突入）
- ユーザーが「今どこ？」と聞いたとき
- **毎ターンの更新はしない**（逆に邪魔になる）
