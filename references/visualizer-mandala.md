# Visualizer: インタラクティブマンダラチャート実装ガイド

このファイルはSKILL.mdのSection 7.1の詳細実装を定義する。
Visualizerの `read_me` → `diagram` モジュールを読み込んでからこのガイドに従うこと。

## 描画仕様

### レイアウト

3×3グリッド。中央にテーマ、周囲8マスにカテゴリ。

```
┌──────────┬──────────┬──────────┐
│  左上     │   上     │  右上    │
│(未発見)   │(カテ1)  │(カテ2)  │
├──────────┼──────────┼──────────┤
│  左       │  中央    │  右      │
│(カテ7)   │[テーマ] │(カテ3)  │
├──────────┼──────────┼──────────┤
│  左下     │   下     │  右下    │
│(カテ6)   │(カテ5)  │(カテ4)  │
└──────────┴──────────┴──────────┘
```

### マスの状態と表示

各マスは3つの状態を持つ:

| 状態 | 枠線 | 背景 | 意味 |
|------|------|------|------|
| 空（empty） | 破線、var(--dc-border-muted) | 透明 | 未検討 |
| 検討中（exploring） | 実線、var(--dc-accent) | var(--dc-accent)の10%透過 | 対話で触れた |
| 充実（filled） | 実線、var(--dc-success) | var(--dc-success)の10%透過 | 内容が十分 |

### インタラクション

各マスはクリック可能。クリック時の動作:

```javascript
// 空マスをクリック → 探索開始
sendPrompt(`「${categoryName}」の観点で探索してください。どんな先行事例や考えるべきポイントがありますか？`)

// 検討中マスをクリック → 深掘り
sendPrompt(`「${categoryName}」についてもっと深掘りしたいです。現在の内容:\n${currentContent}`)

// 充実マスをクリック → 再検討
sendPrompt(`「${categoryName}」を別の角度から見直したいです。現在の内容:\n${currentContent}`)
```

### SVG構造テンプレート

以下の構造でSVGを生成する。CSS変数はread_meモジュールから取得すること。

```
<svg viewBox="0 0 680 500">
  <!-- グリッド背景（薄い線） -->
  <!-- 9マス（rect + text） -->
  <!--   中央マス: テーマ名、太枠、強調色 -->
  <!--   周囲マス: カテゴリ名 + サマリ（2-3行）、状態に応じた枠・背景 -->
  <!--   各マスにonclickでsendPrompt() -->
  <!-- フッター: 「マスをクリックして探索を開始」のヒントテキスト -->
</svg>
```

寸法の目安:
- マス: 約200×145px（間隔含む）
- テキスト: カテゴリ名 14px bold、サマリ 12px regular
- サマリが長い場合は3行で打ち切り（...）
- 中央マスはやや大きめのフォント（16px bold）

### 空欄カウント表示

マンダラの下部に空欄数を表示:
- 「8マス中 N マスが空欄です」
- 全て埋まっている場合: 「すべてのマスに内容があります。レビューに進みますか？」
  - この文言の横にボタン: `sendPrompt('レビューして、空欄や不整合がないか確認して')`

## ドメインテンプレートとの対応

SKILL.md Section 5のテンプレートから、以下のようにマスに配置する:

### ソフトウェア開発
```
[未発見の観点] [ユーザー・アクター] [課題・ゴール]
[非機能要件]   [テーマ名]          [主要機能]
[外部連携]     [UI・操作体験]      [データ・永続化]
```

### 授業設計
```
[未発見の観点] [学習目標]     [学生の前提知識]
[つまずき]     [テーマ名]     [教材・コンテンツ]
[機材・環境]   [時間配分]     [評価方法]
```

### 創作
```
[未発見の観点] [キャラクター] [世界観・設定]
[固有名詞]     [テーマ名]     [プロット・構成]
[対立・葛藤]   [読者体験]     [テーマ・メッセージ]
```

## Markdownフォールバック

Visualizer非対応環境では以下のテーブルで出力:

```markdown
### マンダラチャート: [テーマ名]

|  | 左 | 中央 | 右 |
|---|---|---|---|
| **上段** | ⬜ 未発見の観点 | ✅ ユーザー・アクター: ... | 🔵 課題・ゴール: ... |
| **中段** | ⬜ 非機能要件 | 🎯 **[テーマ名]** | ✅ 主要機能: ... |
| **下段** | ⬜ 外部連携・API | 🔵 UI・操作体験: ... | ⬜ データ・永続化 |

凡例: ✅ 充実 / 🔵 検討中 / ⬜ 空欄

空欄マス: 3/8（未発見の観点、非機能要件、外部連携・API、データ・永続化）
→ 空欄の観点について探索しますか？ どのマスから始めますか？
```
