# スキル発展ロードマップ

## Phase 1: SKILL.md + Visualizer（現在）

**目的**: 自分で使い込んでペインリストを蓄積する。

- ソフトウェア開発テンプレート + マンダラチャート1種類
- Visualizerでインタラクティブマンダラ + ステートビュー
- Obsidian互換md形式でのインポート/エクスポート
- Markdownフォールバック（非Visualizer環境）

**やること**:
- 実際のプロジェクトで繰り返し使う
- ペインリストを蓄積: 「ここにDBが欲しい」「ここのGUIが辛い」「ここの状態復元が不十分」等
- 図法テンプレート追加（フィッシュボーン、SWOT）
- ドメインテンプレート追加（授業設計、創作で実際に使ってみる）

**判断基準**: ペインリストが10項目以上たまったらPhase 2を検討。

## Phase 2: Flask Webアプリ化

**目的**: Phase 1のペインリストを仕様にして永続的なWebアプリを作る。

- Flask（Python）+ SQLite または Obsidian vault直接読み書き
- フロントのSVG/JSはPhase 1のVisualizerコードをそのまま移植（技術資産が腐らない）
- Claude API（claude-sonnet-4-20250514 推奨）で対話エンジン駆動
- ローカルで動かしてブラウザで開く形式
- インタラクティブ図が永続的に保存・復元される

**技術スタック**: Python / Flask / SQLite / HTML+CSS+JS（Visualizerから移植）

## Phase 3: デスクトップアプリ化（任意）

**目的**: Phase 2が十分に動いた後で判断。

- Tauri（Rust + OS標準WebView）: フロントHTML/SVG/JSをそのままラップ。数MB。
- Rust学習が別途進んでいれば検討
- 代替: そのままFlask + ブラウザで十分かもしれない（判断を先送り）

## 将来の機能追加候補

これらはPhase 1のペインリストの結果次第で優先順位が変わる:

- **討論モード**: 3ペルソナ（推進者/批判者/調停者）がNターン議論する「見るだけ発散」。フェーズ横断のモディファイアとして設計
- **ドメイン特化ペルソナ**: ソフトウェア（エンドユーザー/アーキテクト/ビジネス側）、授業（学生/教員/現場のエンジニア）等
- **diff表示**: インポート時に「前回からの変更点」を表示
- **複数人利用**: 学生にこのスキルを使わせる教育シナリオ
- **図法の自動推薦**: Cognitive Fit Theoryに基づく「この話題ならフィッシュボーンが向いています」
- **programjourney.comでの発信**: スキル公開 + 使い方解説 + 思想的背景
- **Obsidianプラグインとの連携**: プラグインAPIの調査

## テンプレート追加の手順

新しいドメインテンプレートを追加するには:

1. 8つの軸（カテゴリ）を定義する
2. 各軸に「探索のヒント」（1-2行の問いかけ）を書く
3. マンダラの配置（上・右上・右…の順）を決める
4. SKILL.md Section 5 に追加する
5. 実際に使ってみて軸の過不足を調整する
